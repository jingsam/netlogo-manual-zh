Bitmap 扩展

Bitmap Extension 做什么？

Bitmap Extension 允许你向绘图和瓦片上操作或导入图像。它提供NetLogo核心原语不提供的特性，例如：scaling, manipulation of different color channels, and width and height reporters.

开始

要导入或者编辑图像必须要在NetLogo中包含bitmap扩展。 

extensions[ bitmap ]

Java虚拟机器的图库决定了支持的图像文件的格式。具体来说就是PNG,JPG,GIF和BMP。PNG是不错的标准选择，几乎哪里都能使用。

如果图像格式支持透明（alpha），这些信息也会被导入。

原语

bitmap:average-color bitmap:channel bitmap:copy-to-drawing bitmap:copy-to-pcolors bitmap:difference-rgb bitmap:export bitmap:from-view bitmap:grayscale bitmap:height bitmap:import bitmap:scaled bitmap:width

bitmap:average-color

bitmap:average-color image

返回一个3个元素的列表，分别是图像的R,G和B，它们是通过加和所有的像素，并利用像素标准化各个组成部分得到的，因此每个组成部分都在0~255之间。

bitmap:channel

bitmap:channel image channel

从图像中提取出alpha, red, green, 或 blue channel。输入的channel应该是一个0-3之间的整数，代表去掉的channel (alpha=0, red=1, green=2, blue=3)。得到的图像是一个代表特定channel的灰度的图像。

bitmap:copy-to-drawing

bitmap:copy-to-drawing image x y

向绘图中导入给定图像，不需要按照给定的像素调整。

bitmap:copy-to-pcolors

bitmap:copy-to-pcolors image boolean

将给定的图像导入pcolors,调整图像，适应整个世界。第二个输入量代表颜色是需要使用NetLogo颜色还是仍然使用RGB颜色。false代表仍然使用RGB颜色。

bitmap:difference-rgb

bitmap:difference-rgb image1 image2

返回一个图像，这个图像的RGB是两个图像RGB的差值。注意第一个图像和第二个图像必须要是相同的宽度和高度，否则会出错。

bitmap:export

bitmap:export image filename

将image 写上 filename。

bitmap:from-view

bitmap:from-view

得到当前视图的图像。

bitmap:grayscale

bitmap:grayscale image

将当前给定图像转为灰度。

bitmap:height

bitmap:height image

返回当前image的高度。

bitmap:import

bitmap:import filename

返回一个LogoBitmap，包含名字为filename的图像。

bitmap:scaled

bitmap:scaled image width height

返回一个新的图像，是原来的image按照给定的width和height调整后的图像。

bitmap:width

bitmap:width image

返回给定image的宽度。
